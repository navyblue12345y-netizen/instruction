## Первый запуск

1. Найдите бота в Telegram и отправьте `/start`
2. Откроется **Главное меню** — нажимайте кнопки для навигации
3. Никаких текстовых команд вводить не нужно, кроме случаев когда бот сам попросит

> ⚠️ Если вы видите "⛔" — у вас нет доступа к этой функции. Обратитесь к суперадмину.

**Навигация по боту:**
- **◀️ Назад** — вернуться на предыдущий экран
- **🏠 Главная** — вернуться в главное меню из любого места
- **❌ Отмена** — отменить ввод текста и вернуться назад

## 📊 Статистика

Показывает сводку по публикациям, сгруппированную по сеткам.

### Статистика за сегодня

Для каждой сетки отображаются все каналы с числами:
- **сег.** — сколько постов вышло сегодня
- **всего** — всего публикаций за всё время
- **очередь** — постов ожидающих публикации
- _(итого сегодня: N)_ — суммарно по сетке

### 📅 За неделю
Количество публикаций за последние 7 дней по каждому каналу.

### 🔗 По источникам
Топ-20 Telegram-каналов источников по количеству использованных постов.

## 📺 Каналы и Сетки

Управление структурой: сетки и входящие в них каналы.

### Что такое сетка

**Сетка** — это группа каналов, объединённых общей темой и расписанием.

Примеры:
- Сетка **"Контент"** — каналы про моду, вязание, дом
- Сетка **"Города России"** — 174 городских новостных канала

Каждая сетка полностью независима:
- своё расписание публикаций
- свои источники
- свои фильтры
- свой режим работы

### Главная страница раздела

Показывает список всех сеток с количеством каналов в каждой.  
Нажмите на сетку чтобы войти в неё.

**Кнопка:** ➕ Новая сетка — создать новую сетку

### Внутри сетки

Показывает список каналов сетки (по 10, с пагинацией). Для каждого канала:

**Строка канала:**
- Название канала — нажмите для просмотра деталей и управления
- Количество источников (✅ если есть, ➕ если нет)
- **⏸** или **▶️** — пауза или возобновление
- **🗑** — удалить канал

**Нажав на название канала** откроется карточка с кнопками:
- **🔗 Источники** — список источников канала
- **⚙️ Фильтры** — настройки медиа, рерайта, стоп-слов для этого канала
- **⏸/▶️** — пауза/возобновление
- **🗑 Удалить канал** — удалить из системы полностью

**Кнопки внизу сетки:**
- **➕ Добавить каналы** — добавить один или несколько каналов в сетку
- **➖ Убрать канал** — убрать канал из сетки (без удаления)
- **🗑 Удалить сетку** — удалить сетку (каналы остаются в системе)

### Создание новой сетки

1. 📺 Каналы & Сетки → **➕ Новая сетка**
2. Введите название сетки
3. Сетка создаётся автоматически с дефолтными настройками
4. Она сразу появится в разделах ⏰ Расписание и 🚫 Фильтры

### Добавление каналов в сетку

1. 📺 → Сетка → **➕ Добавить каналы**
2. Введите ключи каналов через запятую или каждый с новой строки

> Ключ канала — это короткое имя на латинице, например `abakan`, `novosibirsk`

### Удаление канала

При удалении канал полностью убирается из:
- списка каналов системы
- всех сеток
- источников
- настроек

> ⚠️ Удаление необратимо. Перед удалением бот попросит подтверждение.

### Пауза канала

Канал на паузе пропускается при публикации, но все настройки и источники сохраняются.  
Возобновить можно в любой момент нажав **▶️**.

## 🔗 Источники

Источники — Telegram-каналы или RSS-ленты, из которых бот берёт контент.

### Навигация по источникам

Раздел организован по сеткам:

```
🔗 Источники
├── ✅ Контент (3/3)           ← 3 из 3 каналов с источниками
└── ➕ Города России (1/174)   ← 1 из 174 с источниками
```

Нажмите на сетку → откроется список каналов сетки (по 15, с пагинацией).  
Нажмите на канал → список его источников.

### Добавление источников

1. 🔗 → Сетка → Канал → **➕ Добавить**
2. Введите username(ы) Telegram-каналов

**Форматы ввода (все варианты работают):**
```
cityNews_abakan
@cityNews_abakan
cityNews_abakan, another_news
cityNews_abakan
another_news
```
Символ `@` можно не писать — бот добавит сам.

После добавления появятся кнопки:
- **➕ Добавить ещё** — добавить ещё источники для этого канала
- **📋 К каналу** — вернуться к просмотру источников
- **🏠 Главная** — в главное меню

### Просмотр источников канала

Показывает список с разбивкой по типам:
- **telegram** — Telegram-каналы
- **web** — RSS/сайты

Для каждого источника видно количество использованных постов.

### Удаление источника

В списке источников нажмите **🗑 Удалить** → выберите источник → подтвердите.

### ✅ Проверить источники

Кнопка проверяет доступность источника — отвечает ли канал и сколько последних постов доступно.

### Поддерживаемые типы

| Тип | Пример |
|-----|--------|
| Telegram-канал | `city_news_abakan` |
| RSS-лента | `https://abakan-news.ru/feed/` |

## ⏰ Расписание

Каждая сетка имеет **своё независимое** расписание. Изменение одной сетки не влияет на другие.

### Главная страница расписания

Отображает все сетки с текущими настройками:

**Обычный режим:**
```
✅ Контент
  8:00–22:00, каждые 2 ч
  Постов/день: 8 | Медиа: any
  Fetch в 23:00
```

**Живой режим:**
```
✅ Города России
  🔴 Живой режим | fetch каждые 30 мин
  Медиа: any
```

### Настройки сетки

Нажмите **⚙️ [Название сетки]**.

| Параметр | Описание | Диапазон |
|----------|----------|----------|
| ⏱ Интервал публ. | Каждые N часов выходит 1 пост | 1–24 ч |
| 📊 Постов/день | Сколько постов набирать за один fetch | 1–48 |
| 🌅 Начало | С какого часа начинать (МСК) | 0–23 |
| 🌆 Конец | До какого часа публиковать (МСК) | 0–23 |
| 🕐 Час fetch | Когда собирать контент (МСК) | 0–23 |
| 🎬 Тип медиа | Тип медиа для всей сетки | — |
| ⏸/▶️ | Пауза или запуск всей сетки | — |

**Тип медиа для сетки:**
- **🖼+🎥 Любые** — фото и видео
- **📷 Только фото** — только изображения
- **🎥 Только видео** — только видеозаписи
- **📝 Только текст** — без медиавложений
- **✅ Медиа обязательно** — пропускать посты без фото/видео

### 🔴 Живой режим

Для новостных каналов — посты публикуются сразу после появления в источнике.

**Как включить:**
1. ⏰ → ⚙️ [Сетка] → **🔴 Вкл живой режим**
2. Установите интервал проверки (5–180 мин, рекомендуется 30)
3. Нажмите **🔄 Применить сейчас** (обязательно!)

**Как работает:**
```
Каждые N минут:
→ Проверяет новые посты в источниках
→ Есть новые → фильтрует → публикует (до 3 за раз)
→ Нет новых → ничего не делает
```

**Как выключить:** ⏰ → ⚙️ [Сетка] → **🔴 Выкл живой режим** → **🔄 Применить сейчас**

### 🔄 Применить расписание

Кнопка в нижней части страницы расписания.  
Перезапускает движок публикаций с новыми настройками.

> ⚠️ **Обязательно** нажимайте после любых изменений расписания или режима работы.  
> Посты в очереди не теряются при перезапуске.

## 🚫 Фильтры

Определяют какие посты публиковать, а какие пропускать.

### Структура фильтров

```
🚫 Фильтры
├── 🌐 Глобальные     — базовые правила для всей системы
├── ⭐ Приоритет       — слова для приоритетной публикации
├── 🗂 Контент        — правила для сетки "Контент"
│   ├── [Настройки сетки]
│   └── 📋 Каналы сетки (3)
│       ├── moda    → индивидуальные настройки
│       ├── dom     → индивидуальные настройки
│       └── vyazanie → индивидуальные настройки
└── 🗂 Города России  — правила для сетки "Города России"
    ├── [Настройки сетки]
    └── 📋 Каналы сетки (174, пагинация по 15)
        └── abakan → индивидуальные настройки
```

### Приоритет применения

```
Настройки канала  ← наивысший приоритет
        ↓
Настройки сетки   ← если у канала нет своих настроек
        ↓
Глобальные        ← базовый уровень
```

### 🌐 Глобальные фильтры

Применяются ко всем каналам, если не переопределены на уровне сетки или канала.

**Переключатели (вкл/выкл):**

| Переключатель | Описание |
|---------------|----------|
| 📝 Требовать текст | Пропускать посты без текста |
| 🇬🇧 Блок. английский | Блокировать посты на английском языке |
| 🤬 Блок. мат | Блокировать посты с нецензурными словами |
| 📢 Блок. CAPS LOCK | Блокировать посты написанные КАПСЛОКОМ |
| 😶 Блок. только эмодзи | Блокировать посты состоящие только из эмодзи |
| 🔗 Блок. ссылки | Блокировать посты содержащие URL |

**Числовые настройки:**

| Параметр | Описание |
|----------|----------|
| 📏 Мин. символов | Посты короче N символов пропускаются |
| 📐 Макс. символов | Посты длиннее N символов пропускаются (0 = без ограничений) |
| 🔤 Мин. слов | Минимальное количество слов в посте |
| ⏳ Макс. возраст | Не брать посты старше N часов (0 = без ограничений) |

**Списки слов:**

| Список | Описание |
|--------|----------|
| 🚫 Стоп-слова | Посты с этими словами пропускаются |
| ✅ Белый список | Если задан — публикуются ТОЛЬКО посты с этими словами |
| ⚙️ Regex-паттерны | Блокировка по регулярным выражениям (для опытных) |

### ⭐ Приоритетные слова

Посты с этими словами публикуются первыми в очереди.  
Пример: `срочно`, `breaking`, `экстренно`

### 🗂 Фильтры сетки

Нажмите на сетку — откроются настройки применяемые ко всем каналам этой сетки.

**Переключатели сетки:**

| Переключатель | Описание |
|---------------|----------|
| 📷 Только с фото | Публиковать только посты с фото |
| 🎥 Только с видео | Публиковать только посты с видео |
| 🖼 Требовать медиа | Пропускать посты без фото и видео |
| 🔄 Блок. репосты | Блокировать пересланные посты |
| 🗓 Блок. сезонные | Блокировать сезонные поздравления |
| 🛒 Блок. маркетплейсы | Блокировать упоминания WB, OZON и т.д. |
| 🔗 Блок. ссылки | Блокировать посты со ссылками (только в этой сетке) |
| 📢 Блок. CAPS LOCK | Блокировать КАПСЛОК (только в этой сетке) |

Также доступны числовые настройки (мин/макс длина, возраст) и списки слов (стоп-слова, белый список, regex).

**📋 Каналы сетки** — список каналов с индивидуальными настройками.  
Каналы у которых есть индивидуальные настройки отмечены иконкой ⚙️.

### ⚙️ Настройки отдельного канала

🚫 Фильтры → Сетка → **📋 Каналы сетки** → [Канал]

| Настройка | Описание |
|-----------|----------|
| 📹 Тип медиа | Любое / Только фото / Только видео / Только текст / Медиа обязательно |
| ✍️ Рерайт | Вкл / Выкл / Из сетки (наследовать) |
| 📏 Мин. длина | Минимальная длина текста (симв.) |
| 📐 Макс. длина | Максимальная длина текста (симв.) |
| 🚫 Стоп-слова | Список стоп-слов только для этого канала |
| ✅ Белый список | Белый список только для этого канала |
| 🗑 Сбросить | Удалить все индивидуальные настройки канала |

**Управление стоп-словами:**
- **➕ Добавить** — ввести слова через запятую или с новой строки
- **🗑 Удалить** — выбрать слова для удаления из списка

> 💡 **Пример:** канал "Мода" — поставьте "Только видео". Канал "Вязание" — оставьте "Любые медиа". Настройки не влияют друг на друга.

## ✍️ Рерайт

Бот автоматически переписывает текст поста перед публикацией.  
Создаёт короткую вдохновляющую подпись с эмодзи (до 10 слов).

### Глобальные настройки

**▶️ / ⏸ Вкл/выкл** — включить или выключить рерайт для всей системы.

**📝 Промпт** — инструкция для нейросети.

По умолчанию:
> *"Напиши одну вдохновляющую подпись к посту. Одно предложение, максимум 10 слов. Начни со смайлика. Без хештегов и ссылок."*

**🤖 Groq / 🧠 Claude** — выбрать модель для рерайта прямо в этом разделе (быстрое переключение).  
Полное управление провайдерами — в разделе 🤖 Нейросети.

**🔄 Сбросить промпт** — вернуть к заводскому.

### Рерайт для конкретного канала

Настраивается в разделе **🚫 Фильтры → Сетка → Каналы → [Канал] → ✍️ Рерайт**:

| Вариант | Описание |
|---------|----------|
| ⚙️ Из сетки | Использовать настройку от сетки (по умолчанию) |
| ✅ Включить | Рерайт всегда включён для этого канала |
| ❌ Выключить | Рерайт всегда выключен, публикуется оригинальный текст |

> 💡 Для новостных каналов рекомендуется **выключить рерайт** — новости должны выходить с оригинальным текстом.

| **Groq** | ✅ Бесплатно (лимиты) | Хорошее |

### Смена модели

Для некоторых провайдеров можно выбрать конкретную модель:  
🤖 → Провайдер → выберите модель из списка → **✅ Выбрать**

## 🔔 Уведомления

Бот присылает alert если канал долго не публикует посты.

### Настройки

| Параметр | Описание | Рекомендация |
|----------|----------|--------------|
| ▶️/⏸ Вкл/выкл | Включить или отключить все уведомления | — |
| ⏱ Порог молчания | Через сколько часов без постов прислать сигнал | 4–6 ч |
| 🌙 Ночной режим | Сколько часов ночью не беспокоить | 6 ч (с 23:00 до 05:00) |

### Как выглядит уведомление

```
🔕 Молчание канала [Города России]

Канал: Абакан
Последний пост: 02.04 08:00
Молчит: 6 ч
```

### Кто получает уведомления

Все пользователи с ролью **admin** и **superadmin**.

## ⚙️ Система

Управление движком публикаций (content-bot).

### Статус

Показывает запущен ли content-bot. Если статус "❌ Не запущен" — нажмите **▶️ Запустить**.

### 🔄 Перезапустить

Останавливает и запускает content-bot заново с актуальными настройками.

**Когда нужен перезапуск:**
- После изменения расписания
- После включения/выключения живого режима
- Если бот завис и перестал публиковать
- После смены нейросети

> ✅ Посты в очереди при перезапуске **не теряются**.

## 👥 Администраторы

Доступно только суперадмину.

### Роли

| Роль | Что может делать |
|------|-----------------|
| **superadmin** | Полный доступ + управление администраторами |
| **admin** | Все настройки, источники, фильтры, расписание |
| **viewer** | Только просмотр статистики |

### Добавление администратора

1. 👥 Администраторы → **➕ Добавить**
2. Введите Telegram ID пользователя (узнать через @userinfobot)
3. Выберите роль: admin или viewer

### Удаление администратора

В списке нажмите **🗑** рядом с нужным → подтвердите.  
Суперадмина удалить нельзя.

## Как всё работает изнутри

### Обычный режим (например сетка "Контент")

```
23:00 МСК — Ночной сбор (Fetch)
  ↓ заходит в Telegram-каналы из источников
  ↓ берёт N свежих постов на каждый канал
  ↓ проверяет фильтры (реклама, стоп-слова, медиа, длина)
  ↓ переписывает через AI (если рерайт включён)
  ↓ сохраняет в очередь

08:00, 10:00, 12:00 ... 22:00 МСК — Публикация
  ↓ берёт 1 пост из очереди для каждого канала
  ↓ загружает медиа в MAX
  ↓ публикует в канал
```

### Живой режим (например сетка "Города России")

```
Каждые 30 минут:
  ↓ проверяет новые посты в источниках
  ↓ нашёл новые → фильтрует → публикует сразу (до 3 за цикл)
  ↓ ничего нового → ждёт следующей проверки
```

### Защита от дублей

Каждый пост имеет уникальный URL. Бот помнит все увиденные URL за **30 дней** — повторно один и тот же пост не опубликуется.

### Приоритет настроек

```
Настройки канала  ← высший приоритет (переопределяет всё ниже)
      ↓
Настройки сетки   ← применяется если нет настроек канала
      ↓
Глобальные        ← базовый уровень
```

### Независимость сеток

Каждая сетка имеет **явно прописанные** настройки в конфиге. Изменение настроек одной сетки никогда не затрагивает другие.

## Частые вопросы

### Почему бот не публикует?

Проверьте по порядку:
1. ⚙️ Система — запущен ли content-bot
2. ⏰ Расписание → сетка — активна ли сетка (нет ⏸)
3. 📺 → Сетка → канал — не на паузе ли канал
4. 🔗 → Сетка → канал — есть ли источники (не ➕)
5. 📊 Статистика → канал — есть ли посты в очереди

### Посты выходят с коротким непонятным текстом

Причина: рерайт включён, но оригинальный текст поста был слишком коротким.  
Решение: 🚫 Фильтры → Сетка → Каналы → [Канал] → ✍️ Рерайт → **❌ Выключить**

### Выходят посты с фото, хотя нужно только видео

1. 🚫 Фильтры → Сетка → 📋 Каналы → [Канал] → 📹 Тип медиа → **🎥 Только видео**
2. Изменение применяется к новым постам при следующем fetch (23:00)
3. Текущие неправильные посты в очереди очистятся автоматически при ночном сборе

### Как включить живой режим для новостей?

⏰ Расписание → ⚙️ [Сетка] → **🔴 Вкл живой режим** → установить интервал (например 30 мин) → **🔄 Применить сейчас**

### Изменил расписание — ничего не изменилось

Нажмите **🔄 Применить расписание** в разделе ⏰.  
Все остальные настройки (фильтры, источники, рерайт) применяются **сразу**, без перезапуска.

### Как добавить новую тематическую сетку?

1. 📺 → **➕ Новая сетка** → введите название
2. ⏰ → ⚙️ [Сетка] → настройте расписание → **🔄 Применить**
3. 📺 → Сетка → **➕ Добавить каналы** → введите ключи каналов
4. 🔗 → Сетка → добавьте источники для каждого канала
5. 🚫 → Сетка → настройте фильтры и тип медиа при необходимости

### Бот завис, ничего не публикует

⚙️ Система → **🔄 Перезапустить**

### Почему некоторые посты пропускаются?

Причины записываются в базу данных:

| Код | Причина |
|-----|---------|
| `media_file_missing` | Медиафайл не удалось скачать с сервера |
| `no_media_filter=video` | В посте нет видео, а канал настроен на "Только видео" |
| `no_content` | Пустой пост — нет ни текста, ни медиа |
| `max_api_error` | Ошибка при отправке в MAX (временная проблема) |
| `stale_pending` | Пост устарел (>24ч в очереди, заменён новым) |
